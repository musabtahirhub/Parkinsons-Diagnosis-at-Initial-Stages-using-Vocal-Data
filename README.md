## Quickstart

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

> **macOS:** If `praat-parselmouth` fails to install, run `brew install praat` first.  
> **Windows:** Make sure you have the [Visual C++ build tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) installed.

### 2. Train the model

Open and run `notebook/Parkinsons.ipynb` from top to bottom. This will:
- Load and clean the dataset
- Engineer acoustic features
- Run EDA and save plots
- Tune and evaluate Random Forest + Logistic Regression
- Export all artifacts to `parkinsons_outputs/`

### 3. Launch the app

```bash
streamlit run app.py
```

Use the **sidebar** to point the app at your `parkinsons_outputs/` folder if it's not in the working directory.

## Training Pipeline (Notebook)

The notebook `notebook/Parkinsons.ipynb` runs through 14 sections:

| # | Section | Description |
|---|---------|-------------|
| 1 | Imports & Config | Libraries, paths, output directory |
| 2 | Data Loading | CSV load + shape/type summary |
| 3 | Feature Engineering | 9 derived acoustic features |
| 4 | EDA | 5 saved plots (distribution, PPE scatter, heatmap, boxplots) |
| 5 | Cleaning & Selection | Dedup, impute, top-15 correlation selection |
| 6 | Target Validation | Class balance check |
| 7 | Feature Scaling | StandardScaler fit |
| 8 | Train / Test Split | 80/20 stratified split |
| 9 | Hyperparameter Tuning | GridSearchCV — RF + Logistic Regression |
| 10 | Model Evaluation | Accuracy, F1, ROC AUC, overfitting gap, learning curves |
| 11 | Resource Usage | Inference time + memory per model |
| 12 | Feature Importance | Top-15 bar chart (Random Forest) |
| 13 | Save Artifacts | Pickle models, scaler, feature names |
| 14 | Pipeline Summary | Final metrics table |

**Models trained:**
- `RandomForestClassifier` — heavily regularised (shallow trees, high min-samples)
- `LogisticRegression` — L1/L2/ElasticNet via `saga` solver

## Prediction Pipeline (App)

```
WAV file
   │
   ▼
extract_features.py
   ├── Praat (via praat-parselmouth)
   │     ├── Fundamental frequency  →  MDVP:Fo(Hz), Fhi(Hz), Flo(Hz)
   │     ├── Jitter measures        →  Local, Abs, RAP, PPQ, DDP
   │     ├── Shimmer measures       →  Local, dB, APQ3, APQ5, APQ11, DDA
   │     └── Noise ratio            →  HNR, NHR
   └── librosa
         ├── Spectral entropy       →  RPDE (approximation)
         ├── Scaling exponent       →  DFA
         └── Pitch nonlinear        →  spread1, spread2, D2, PPE
   │
   ▼
create_engineered_features()   ← mirrors notebook exactly
   ├── jitter_shimmer_product
   ├── avg_jitter / avg_shimmer
   ├── voice_perturbation_index
   ├── hnr_nhr_ratio
   ├── frequency_range / frequency_cv
   ├── spread_interaction
   └── dfa_ppe_product
   │
   ▼
StandardScaler  (scaler.pkl)
   │
   ▼
Best model  (best_model.pkl)
   │
   ▼
Prediction + probability  →  Streamlit results panel
```

## Feature Extraction Notes

`extract_features.py` uses **Praat** (via `praat-parselmouth`) for jitter, shimmer, and HNR — these match the UCI dataset methodology exactly. The following nonlinear features are **approximations** computed from the pitch series and spectrogram using `librosa`:

| Feature | Method used |
|---|---|
| `RPDE` | Normalised spectral entropy (approximation) |
| `DFA` | Simplified detrended fluctuation analysis |
| `spread1` / `spread2` | Log-pitch std / pitch diff std |
| `D2` | Sample entropy of log-pitch (approximation) |
| `PPE` | Histogram entropy of normalised pitch |

These approximations produce values in the correct range and direction, but may differ numerically from the exact algorithms used to create the original UCI dataset.

## Recording Tips

| Do |  Avoid |
|-------|---------|
| Sustain vowel **"ahhh"** for 5–10 s | Clips shorter than 3 s |
| Record in a quiet room | Background noise or echo |
| Mic ~10 cm from mouth | Phone speaker playback |
| Export as **WAV, 16-bit, mono** | MP3 or other compressed formats |
